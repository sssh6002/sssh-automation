# 來文電子檔處理方式

收到公文後自動下載的電子檔的處理方式

## 輸出總覽（重要）

對「每個」公文目錄處理時，必須**同時**產出兩個檔（皆寫進該目錄）：
1. `公文主檔名內容.txt`（純文字、保留指定欄位 — 規則見 §存純文字檔完整內容）
2. `公文主檔名總結.<LLM模型名>.md`（含分類標記 + LLM 摘要 — 規則見 §公文主檔總結方式）

「略過規則」分別獨立：
- 已有 `公文主檔名內容.txt` → 該次只略過 `.txt` 工作，仍要產出 `.md`
- 已有 `公文主檔名總結.<LLM模型名>.md` → 該次只略過 `.md` 工作，仍要產出 `.txt`
- 兩個都已存在 → 整目錄略過（用程式 prompt 指定的「格式 2 SKIP」）

## 電子檔組成

本目錄位於資料夾名:MW開頭數字 之下
公文主檔名:數字_數字[A~Z].pdf，其中[A~Z]可省略(即「數字_數字.pdf」亦為合法主檔名)。其他檔案一律不處理。
### 存純文字檔完整內容
完整保留
*發文日期
*發文字號
*主旨
*說明
*附件
其他公文內容一律不要。
*再將此檔之以上內容存成檔名為:公文主檔名內容.txt。
*若發現該目錄已有此電子檔，則略過此工作

### 公文主檔總結方式

完整保留
*發文日期
*發文字號
*主旨

其他公文內容一律不要

*若主旨或說明內有以下相關字詞(LLM協助辨識)，則作相對應分類後。
*「相關字詞」一個欄位可以有一個以上的字詞的可能，以OR區隔。 
*將「存查分類」置於總結檔的最上方一行，開頭加"#存查分類:"，如#存查分類:資安
*於輸出分類文字後方，空一格後接著輸出其對應檔號，對應關係為:研習OR競賽=03750401，資安=03750402，資訊教學=03750403，設備=03750404。
*將「承辦文字」置於「存查分類」下一行，開頭加"##"，如:##不參加 
*將「承辦方式」置於「承辦文字」下一行，開頭加"###"，如:###陳會 
*以下為對應表，愈上方的愈優先，若有衝突，以上方為準

#### 再以LLM做總結
*說明裏的內容，字數需(<原總說明15%)OR(<250字)。
*總結以數字開頭，分條呈現
*存檔於本目錄，檔名為:公文主檔名總結.現在使用的LLM模型名.md
*若發現該目錄已有總結檔，則略過總結工作

故會產生兩個檔案:公文主檔名內容.txt，公文主檔名總結.現在使用的LLM模型名.md
（兩個檔在同一次 LLM 回應中一併輸出 — 詳見本檔最上方「輸出總覽」）
